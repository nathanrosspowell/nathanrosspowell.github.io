First!

Posted: 2014/12/29 13:23

Hello, blogosphere!

I have re-re-made my personal website, again, again! This time I have remade everything using the assemble.io framework which is based on node.js.

The linux laptop that I was developing my website on reached the end of it's life and setting up the project to work on Windows (temporarily) was going so well... The templates had reached the point where I was unsure what world I was living on... So, the re-write happened!

Version 1 - Django

The very first time I tried to make this website I had very little experience outside of editing raw HTML. With my programming knowledge mainly based in C++ and Python, I did some research and settled on using the Django framework.

After going through all of tutorials and setting up a basic blog I fell at the hurdle of getting the thing hosted. Whatever year it was when I attempted this, combined with my limited knowledge of all things server side, I found it hard to get a free/cheap solution for hosting the python back end.

Version 2 - Flask

Having a database, users, online interface for publishing articles, etc. was all a bit much for a small toy website like this. With some more research I found out about static websites and that lead me to the Frozen Flask project which generated static HTML files for a app made with the Flask micro framework.

Flask seemed to fit my use case better than Django and with a little work the static website output was easy to copy across to the GitHub Pages branch of the project.

With the ability to push the website straight to the free hosting I managed to get more structure and 'style' than Version 1. A lot of iteration was done on this version and as I learned about all things web.

Version 2.5 - Grunt

As I made other toy websites such as Interface Zero and Presentations I leverages the power of Node.js and Grunt. Grunt replaced all of the shell scripts that I had made for automating the tasks of running the static site generation. As that was so easy to do, I also got packages to enable me to prettify and minimize parts of the website. I also got a very nice package grunt-gh-pages which simplified pushing the dist files to the server.

Version 3 - assemble.io

When I lost the dev environment that I set up on a laptop to work with the Flask project I stopped updating it. When I wanted to make some improvements to the site I jumped into the code (specifically the template for the navbar) and I was completely lost! A classic programmers error - expecting that the code would make sense in the future!

Seeing as there is very little traffic to the site I decided to just re-do the whole thing from scratch.

With node installed, there are only a few global package installs and then the website can be set up to run. I currently have the dev environment running on Linux and Windows 7. The built in task grunt server allows me to make 'live changes' to the locally server files greatly improving the work flow from my hashed together Flask and Grunt combo in Version 2.

The Future

assemble.io is still being developed. In the future I will upgrade some of the code to make use of plugins written by the community and maybe write some more detailed write ups on how I get assemble.io to work for me.
